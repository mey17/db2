NoSQL database is a non-relational Data Management System, no fixed schema. No joins, easy to scale.
Distributed data stores with humongous data storage needs.
• Key-value pair
• Column-oriented graph
• Graphs based
• Document-oriented


NoSQL databases can be categorized based on their underlying data model:

Key-Value Stores (like a dictionary).
    Use Case: Session management, caching, or user profiles.

Column-Oriented Databases (columns rather than rows, optimized for analytical queries)
    Use Case: Real-time analytics, big data processing.

Graph Databases (nodes and edges, representing relationships)
    Use Case: Social networks, recommendation engines, or fraud detection.

Document-Oriented Databases (documents JSON, BSON)
    Use Case: Content management systems, catalogs, or product information storage.

Data Warehousing (DW) - large centralized repository (from one or more data sources), supports analytics and reporting.
Merges information coming from different sources into one comprehensive database.
